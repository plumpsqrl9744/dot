# dot

React와 TypeScript를 활용하여 빠르게 구축된 Electron 데스크톱 애플리케이션입니다.

## 💻 추천 IDE 환경

- 최적의 개발 경험을 위해 [VSCode](https://code.visualstudio.com/) 사용을 권장합니다.
- 코드 품질 관리를 위해 [ESLint](https://marketplace.visualstudio.com/items?itemName=dbaeumer.vscode-eslint)와 [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) 확장을 설치해 주세요.

## 🚀 프로젝트 설정 및 실행 방법

### 패키지 설치

이 프로젝트는 패키지 매니저로 Yarn을 사용합니다. 아래 명령어로 필수 의존성을 설치해 주세요.

```bash
$ yarn
```

### 개발 모드 실행

개발 중에는 아래 명령어를 사용하여 Hot-Reload가 지원되는 환경을 켤 수 있습니다.

```bash
$ yarn dev
```

### 상용 빌드

각 운영체제에 맞는 실행 파일로 패키징할 수 있습니다.

```bash
# Windows 전용 빌드
$ yarn build:win

# macOS 전용 빌드
$ yarn build:mac

# Linux 전용 빌드
$ yarn build:linux
```

## 🛠 주요 기술 스택

- **데스크톱 환경**: [Electron](https://www.electronjs.org/)을 기반으로 운영체제 네이티브 기능을 활용합니다.
- **프론트엔드 프레임워크**: 최신 [React 19](https://react.dev/)를 사용하여 선언적이고 풍부한 사용자 경험(UX)을 제공합니다.
- **언어**: [TypeScript](https://www.typescriptlang.org/)를 도입하여 정적 타입 체크를 통한 안정성을 확보했습니다.
- **로컬 저장소**: SQLite를 이용한 서버리스 로컬 데이터베이스를 구축하여 클라우드 의존성 없이 오프라인에서도 데이터를 저장하고 유지할 수 있습니다.
- **번들링 및 빌드 도구**: [Electron Vite](https://electron-vite.org/)를 적용하여 빌드 시간을 단축하고 HMR(초고속 모듈 교체)을 지원합니다.

## 📝 애플리케이션 주요 특징

- **완벽한 데스크톱 통합**: 브라우저를 벗어나 **Electron.js** 기반의 독립된 데스크톱 애플리케이션 환경 내에서 **React.js**가 유려하게 동작합니다.
- **강력하고 빠른 오프라인 데이터베이스**: 사용자의 컴퓨터 로컬 디렉토리에 **SQLite**를 직접 연동하여 중요 정보를 안전하게 보관합니다. 외부 서버 의존성을 없애 완벽한 오프라인 환경을 구현하며 극도로 빠른 조회 성능을 자랑합니다.

## 💡 번외 기획: P2P 웹 Todo 앱을 구축하신다면?

만약 현재 프로젝트(Electron) 외에, **React나 Next.js 기반의 웹 환경**에서 회사 동료들과 Todo를 실시간으로 공유하는 **P2P 애플리케이션**을 새롭게 기획 중이시라면 다음의 기술 스택 도입을 강력히 추천해 드립니다.

서버를 통한 중앙 집중식 데이터 통신 없이, 브라우저끼리 직접 데이터를 동기화하는 데에 탁월함을 발휘합니다.

- **데이터 기반 및 상태 관리**: **Yjs**
  - 충돌 방지(CRDT) 알고리즘을 통한 완벽한 실시간 분산 데이터 동기화 라이브러리입니다.
- **네트워크 계층 연결**: **y-webrtc**
  - WebRTC 기술을 활용하여 사용자 브라우저 간의 직접적인 (Peer-to-Peer) 네트워크 경로를 연결해 주는 통신 모듈입니다.

Gemini의 응답
한비로(Hanbiro) 그룹웨어의 메일을 자동으로 읽어 로컬 AI로 To-do List를 만드는 프로젝트는 업무 효율을 획기적으로 높일 수 있는 아주 좋은 아이디어입니다.

로컬 환경에서 가볍고 무료로 사용할 수 있는 AI 모델과 전체적인 개발 아키텍처를 정리해 드릴게요.

1. 추천 로컬 AI 모델 (LLM)
   로컬 컴퓨터(일반적인 PC/노트북)에서 돌리려면 매개변수(Parameter)가 적으면서도 한국어 능력이 검증된 모델이 필요합니다.

모델명 특징 추천 이유
Llama-3-8B (Instruction) 현재 가장 범용적인 오픈소스 모델 텍스트 요약, 중요도 판단 성능이 매우 뛰어남
Gemma-2-9B 구글에서 공개한 오픈소스 모델 한국어 이해도가 높고 문장 요약이 깔끔함
EEVE-Korean-10.8B 한국어 특화 파인튜닝 모델 한국어 번역 및 요약에 있어 가장 자연스러운 성능을 보임

2. 개발 방식 및 아키텍처
   이 시스템은 크게 [메일 수집] -> [AI 분석] -> [To-do 저장] 3단계로 구성됩니다.

Step 1: 메일 데이터 가져오기 (Hanbiro API/IMAP)
한비로는 보통 기업용 솔루션이므로 API를 제공하거나 표준 IMAP/POP3 프로토콜을 사용합니다.

방법: Python의 imaplib 라이브러리를 사용하여 실시간 혹은 주기적으로 '받은 편지함'을 체크합니다.

필터링: 모든 메일을 분석하면 리소스 낭비이므로, 읽지 않은 메일이나 특정 키워드가 포함된 메일만 추출합니다.

Step 2: 로컬 AI 구동 환경 구축 (Ollama)
로컬에서 AI를 가장 쉽고 가볍게 돌리는 방법은 Ollama를 사용하는 것입니다.

Ollama 설치: ollama.ai에서 다운로드 후 설치.

실행: 터미널에서 ollama run llama3 (또는 gemma2) 명령어로 즉시 모델 가동.

활용: Python 코드에서 Ollama API를 통해 텍스트를 던지고 결과(JSON 형태)를 받습니다.

Step 3: AI 프롬프트 구성 (요약 및 중요도)
AI에게 단순히 "정리해줘"라고 하기보다 JSON 구조로 답변을 달라고 요청해야 프로그램이 이해하기 쉽습니다.

Prompt 예시:
"다음 이메일 내용을 분석해서 1. 작업 내용, 2. 마감 기한, 3. 중요도(상/중/하)를 JSON 형식으로 요약해줘. 언어는 한국어로 번역해서 출력해."

3. 기술 스택 요약
   언어: Python (자동화 및 AI 연동에 최적)

AI 추론 엔진: Ollama (로컬 서버 역할)

프레임워크: LangChain (메일 텍스트를 AI에게 효율적으로 전달하기 위함)

데이터베이스 (Todo List): SQLite (파일 기반의 가벼운 DB) 또는 Notion API 연동

4. 개발 로직 흐름도
   Python 스크립트 실행: 5분마다 한비로 메일 서버 접속.

새 메일 발견: 메일 제목과 본문 텍스트 추출.

AI 분석 요청: 로컬 Ollama 서버에 텍스트 전달.

AI 작업: 영문 메일 번역 -> 핵심 요약 -> To-do 도출 -> 중요도 산출.

결과 저장: 분석된 데이터를 To-do List DB에 인서트(Insert).

알림(선택): 텔레그램 봇 등을 통해 "새로운 할 일이 등록되었습니다" 알림 발송.

1. IMAP/POP3 프로토콜 활용 (가장 추천)
   대부분의 기업용 그룹웨어는 외부 메일 클라이언트(아웃룩, 스마트폰 메일 앱 등) 연동을 위해 IMAP 기능을 제공합니다. 파이썬(Python)의 기본 라이브러리인 imaplib를 사용하면 프로그램이 직접 한비로 서버에 접속해서 새 메일을 가져올 수 있습니다.

작동 방식: 프로그램이 1분이나 5분마다 한비로 서버에 "새로 온 메일 있어?"라고 물어보고(Polling), 있다면 내용을 긁어옵니다.

장점: 별도의 복잡한 API 권한 신청 없이 계정 정보(ID/PW)와 서버 주소만 알면 바로 개발 가능합니다.

코드 예시 (Python):

```Python
import imaplib
import email

# 한비로 IMAP 서버 설정 (서버 주소는 한비로 설정 페이지에서 확인 필요)
mail = imaplib.IMAP4_SSL('mail.hanbiro.com')
mail.login('본인계정', '비밀번호')
mail.select('inbox')

# 읽지 않은(Unseen) 메일 검색
status, response = mail.search(None, '(UNSEEN)')
```

## 실시간성을 높이는 팁

매번 수동으로 프로그램을 실행할 수는 없으니, 다음과 같은 방식을 병행합니다.

스케줄러(Scheduler): 파이썬의 schedule 라이브러리나 윈도우의 '작업 스케줄러'를 이용해 5분마다 스크립트가 자동으로 돌게 만듭니다.

백그라운드 실행: 프로그램을 끄지 않고 트레이 아이콘 형태로 계속 띄워두어 메일이 올 때마다 AI 분석을 시작하게 합니다.
